# Security Policy

## Supported Versions

Only the latest release (the `main` branch) is actively supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

---

## Reporting a Vulnerability

We take the security of the **Clinical Cohort Query Mapper** seriously. If you believe you have found a security vulnerability, please do NOT open a public issue. Instead, report it directly to the engineering team.

### How to Report
Please email details of the vulnerability to **[engineering@srotas.ai](mailto:engineering@srotas.ai)**. 

To help us investigate, please include:
- A description of the issue.
- Step-by-step instructions (or a proof-of-concept script) to reproduce the vulnerability.
- Potential impact of the vulnerability.

### What to Expect
- We will acknowledge receipt of your vulnerability report within 48 hours.
- We will work to investigate and address the issue as quickly as possible.
- Once fixed, we will publish a patch release and notify you.
